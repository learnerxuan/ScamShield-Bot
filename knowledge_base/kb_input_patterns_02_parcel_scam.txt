SECTION 2: COMMON PARCEL-SCAM WORDING PATTERNS
================================================================================

FAILED DELIVERY / MISSED PARCEL
- "delivery failed"
- "missed parcel"
- "unable to deliver"
- "reschedule delivery"
- "parcel will be returned"

ADDRESS OR INFORMATION UPDATE
- "invalid address"
- "update your address"
- "confirm recipient information"
- "update delivery details"

FEE OR CUSTOMS PAYMENT
- "customs fee"
- "redelivery fee"
- "postage due"
- "release fee"
- "pay now to receive parcel"

UNEXPECTED PARCEL
- "your parcel is waiting"
- "undelivered package"
- "package is on hold"
- user has no matching order history

================================================================================
SECTION 3: LINK, APP, AND ESCALATION PATTERNS
================================================================================

- tracking or reschedule link:
  - "track parcel"
  - "reschedule now"
  - "update shipping"
- app or APK install:
  - "download app"
  - "install tracking app"
  - "APK"
- courier impersonation:
  - message claims to be from Pos Malaysia, DHL, J&T, FedEx, Ninja Van
- customs or police escalation:
  - "parcel held at customs"
  - "illegal items found"
  - "police investigation"
  - "avoid arrest"
  - "transfer to safe account"

================================================================================
SECTION 5: POST-INCIDENT INDICATORS
================================================================================

Escalate immediately if user says they:
- already clicked the link
- already paid the fee
- already entered banking or card details
- already installed the app / APK
- already spoke to the caller and were told to transfer money
- already transferred money to a so-called safe account

================================================================================
SECTION 6: RULE-BASED USE IN THE BOT
================================================================================

Recommended flow:
1. User pastes message or link.
2. Bot extracts parcel-scam indicators from the pasted content.
3. Bot asks follow-up questions:
   - Were you expecting this parcel?
   - Did it ask for payment?
   - Did it ask you to install an app?
   - Did it mention police, customs, or a safe account?
   - Did you already pay or install anything?
4. Bot calculates final risk from both the pasted content and user answers.

================================================================================
SECTION 7: LIMITATIONS
================================================================================

Do not claim these from pasted text or visible link alone:
- that a URL is definitely malicious
- that a courier tracking number is fake without checking the official courier site
- that an APK actually contains malware without technical inspection
- that a police/customs claim is false without independent verification

Therefore:
- The feature must be framed as a preliminary rule-based assessment.
- The bot should not behave like a live courier lookup system or malware scanner.
