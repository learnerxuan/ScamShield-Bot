SECTION 2: COMMON FAKE BANK-ALERT WORDING PATTERNS
================================================================================

SECURITY ALERT / ACCOUNT PROBLEM
- "suspicious transaction"
- "unusual login"
- "your card was used"
- "security verification required"
- "account suspended"
- "account locked"

CREDENTIAL REQUEST
- "confirm OTP"
- "provide PIN"
- "verify password"
- "share TAC code"
- "confirm account details"

SAFE ACCOUNT / MOVE YOUR MONEY
- "move your money"
- "transfer to safe account"
- "secure account"
- "temporary holding account"

URGENT CALL TO ACTION
- "call immediately"
- "urgent action required"
- "within 24 hours"
- "respond now"

================================================================================
SECTION 3: LINK, HOTLINE, AND IMPERSONATION PATTERNS
================================================================================

- bank-like login or verification link
- suspicious hotline or callback number in the message
- bank or regulator impersonation
- fake app or off-store install instruction

Examples:
- "click here to verify transaction"
- "call this number immediately"
- "fraud team from your bank"
- "install security app to protect account"

- requestsSafeAccountTransfer
- requestsAppInstall
- hasUrgency

================================================================================
SECTION 5: POST-INCIDENT INDICATORS
================================================================================

Escalate immediately if user says they:
- already shared OTP or TAC
- already shared PIN or password
- already clicked the login or verification link
- already called the suspicious hotline
- already transferred money to a safe account
- already installed an app from the message

================================================================================
SECTION 6: RULE-BASED USE IN THE BOT
================================================================================

Recommended flow:
1. User pastes message or link.
2. Bot extracts bank-alert indicators from the pasted content.
3. Bot asks follow-up questions:
   - Did it claim to be from your bank?
   - Did it ask for OTP, PIN, or password?
   - Did it give a suspicious link or hotline?
   - Did it ask for a safe-account transfer?
   - Did you already share any banking details?
4. Bot calculates final risk from both the pasted content and user answers.

================================================================================
SECTION 7: LIMITATIONS
================================================================================

Do not claim these from pasted text or visible link alone:
- that a bank website is definitely fake without verification
- that a phone number is definitely malicious without independent checking
- that a message is definitely from a real bank just because the wording looks polished
- that the user's account is compromised without evidence

Therefore:
- The feature must be framed as a preliminary rule-based assessment.
- The bot should not behave like a live bank-authentication or telecom-verification tool.
