SECTION 1: PURPOSE
================================================================================

This file supports the planned "paste suspicious message or link" feature for fake
bank-alert and bank-impersonation scams. It focuses on text patterns, visible URL
patterns, and impersonation cues that can be extracted from user-pasted content.

This file should be used to:
- detect fake bank-alert wording patterns
- detect fake hotline and safe-account language
- detect credential-harvesting cues
- trigger follow-up questions
- strengthen bank-alert risk scoring

================================================================================
SECTION 2: SUPPORTED INPUT TYPES
================================================================================

SUPPORTED:
- SMS bank-alert text
- WhatsApp/chat text claiming to be from a bank
- email text
- visible banking-related URL or hotline in pasted content

NOT SUPPORTED BY THIS FILE ALONE:
- checking live bank systems
- verifying whether a caller ID was spoofed
- confirming website legitimacy beyond visible text cues
- inspecting mobile apps or attachments

================================================================================
SECTION 3: COMMON MESSAGE WORDING PATTERNS
================================================================================

PATTERN GROUP A - SECURITY ALERT / ACCOUNT PROBLEM
Examples of wording cues:
- suspicious transaction detected
- unusual activity
- account suspended
- account frozen
- secure your account now
- verify transaction immediately

Interpretation:
- Strong cue when message also contains a link or callback number.

PATTERN GROUP B - CREDENTIAL REQUEST
Examples of wording cues:
- provide OTP
- confirm PIN
- verify password
- update banking details
- confirm card information

Interpretation:
- High to Critical cue.
- Legitimate banks do not request these through such channels.

PATTERN GROUP C - CALLBACK / HOTLINE REDIRECTION
Examples of wording cues:
- call this number immediately
- contact fraud department now
- urgent hotline
- call support to stop transaction

Interpretation:
- High-risk cue when the number is embedded inside the suspicious message.

PATTERN GROUP D - SAFE ACCOUNT LANGUAGE
Examples of wording cues:
- transfer your money to a safe account
- secure account
- temporary holding account
- move funds to protect them

Interpretation:
- Critical cue.
- This is a core bank-impersonation fraud pattern.

================================================================================
SECTION 4: SUSPICIOUS URL / LINK PATTERNS VISIBLE FROM TEXT
================================================================================

VISIBLE LINK CUES:
- login link sent in security alert message
- domain that looks like the bank but is slightly different
- fake "secure" or "verify" words added to bank name
- shortened or masked URLs
- visible mismatch between bank name and domain

BOT-USEFUL EXTRACTION FLAGS:
- containsLoginLink
- containsBankBrand
- visibleBankDomainMismatch
- containsHotlineNumber
- containsSafeAccountLanguage
- usesShortenedUrl

================================================================================
SECTION 5: SENDER / IMPERSONATION PATTERNS
================================================================================

SENDER CUES:
- message claims to come from bank or regulator
- sender provides a number different from official card/hotline channels
- caller claims urgency to bypass normal verification
- message uses institutional tone to force compliance

COMMON IMPERSONATION TARGETS IN SOURCE MATERIAL:
- banks
- BNM
- bank fraud department / security unit

================================================================================
SECTION 6: EXPLICIT EXTRACTABLE INDICATORS FOR THE BOT
================================================================================

The parser should attempt to detect:

- claimsBankIdentity
- claimsRegulatorIdentity
- mentionsSuspiciousTransaction
- mentionsAccountSuspension
- asksForOtp
- asksForPin
- asksForPassword
- asksForBankingDetails
- containsLoginLink
- containsHotlineNumber
- visibleBankDomainMismatch
- asksToCallNumberInMessage
- asksForSafeAccountTransfer
- hasUrgencyLanguage

Suggested use:
- use these indicators to classify likely fake bank alert
- still ask follow-up questions before final assessment

================================================================================
SECTION 7: POST-INCIDENT INDICATORS
================================================================================

These signals indicate likely compromise or urgent danger:

- user already clicked the login link
- user already shared OTP
- user already shared PIN or password
- user already called the hotline in the message
- user already transferred money to a "safe account"

Suggested rule handling:
- shared OTP / PIN / password = CRITICAL
- safe-account transfer = CRITICAL
- clicked suspicious login link = HIGH, escalate with follow-up

================================================================================
SECTION 8: LIMITATIONS
================================================================================

The following cannot be safely inferred from pasted text or visible link alone:

- whether a hotline number truly belongs to a fraudster unless cross-checked
- whether the message came from a spoofed sender ID
- whether the linked site performs backend credential harvesting
- whether a transaction alert is genuine without checking the real bank account
- whether a bank app or webpage is technically compromised

This file supports PRELIMINARY RULE-BASED ASSESSMENT only.
